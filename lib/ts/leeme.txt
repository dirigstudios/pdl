TAD TS en C++

Este fichero contiene una implementación, realizada en lenguaje C++, de una clase con un tipo abstracto de datos (TAD) que representa una tabla de símbolos.

Los ficheros adjuntos son:

- leeme.txt: este fichero
- *.h: ficheros de cabecera necesarios
- *.cpp: ficheros fuente para compilar junto a los ficheros que usen este TAD
- pruebas.zip: fichero comprimido que contiene una serie de ejemplos de utilización
- Manual.doc: manual del usuario en MS-Word

La librería se proporciona tal cual (con el código fuente para su libre uso y modificación) y es posible que no esté libre de errores, por lo que si se detecta alguno y se corrige el código, se agradecería que se notificara al webmaster de Compiladores.

Lista de versiones y errores que corrigen:
==========================================

19-11-2002
Se incluye en el paquete el fichero de cabecera "except.h" del compilador de C++ de Borland, por si en el entorno de programación utilizado resulta necesario. La librería equivalente de Visual C++ es "excpt.h".
Se modifican los ejemplos, pues incluían un fichero de cabecera no necesario.
9-01-2003
Se modifica el código para hacerlo compatible con la herramienta Microsoft Visual C++ versión 6.0: eliminación de las sentencias typeid (sustitución por un nuevo método de la clase CAtributo DarTipo), eliminación de las sentencias dynamic_cast(sustitución por un casting simple) y tratamiento 
de las excepciones mediante una clase de excepciones existente en ambas herramientas (inclusión de unas directivas de compilación). 
Se adaptan las pruebas para que funcionen en Microsoft Visual C++ versión 6.0. 
21-02-03
Se modifica el código para hacerlo compatible con el compilador g++ de GNU en Linux.
24-02-03
Se cambian los ficheros .h por una equivocación de versiones.
18-03-03
Se añade información al manual sobre el modo de compilación en varios entornos y sistemas.
Se modifican algunos métodos que sólo ocupan una línea utilizando inline en el .h y eliminándolos del .cpp.

Tras 9 meses en los que no se ha recibido ningún informe de errores, se deciden terminar las pruebas beta y liberar la versión final.
